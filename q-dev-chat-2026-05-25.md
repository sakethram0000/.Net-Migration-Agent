## What Patterns Exist in Multi-Agent Systems

There are several well-known multi-agent architecture patterns:

| Pattern | What it means |
|---|---|
| **Sequential Pipeline** | Agents run one after another, fixed order, no decisions |
| **Supervisor / Orchestrator** | One central agent controls all others, makes decisions |
| **Hierarchical** | Multiple levels — supervisor controls sub-supervisors who control agents |
| **Blackboard** | Shared memory space, agents read/write to it, no central controller |
| **Peer-to-Peer / Collaborative** | Agents talk directly to each other, no central controller |
| **ReAct (Reason + Act)** | Agent reasons about what to do, acts, observes result, reasons again |
| **Plan and Execute** | Planner agent creates a plan first, executor agents carry it out |
| **Swarm** | Many small agents working in parallel, emergent behavior |

```
        Top Supervisor
       /              \
  Supervisor A    Supervisor B
   /    \            /    \
Ag1    Ag2         Ag3    Ag4
```

Multiple levels of supervisors. Each supervisor manages a group of agents. Top supervisor manages supervisors.

```
AgentA → AgentB → (if X) → AgentC
                → (if Y) → AgentD
```
A custom defined flow with conditional routing. Not fully dynamic but not fully fixed either.

---

### 5. Plan and Execute (Planner + Executor)
```
Planner Agent → creates plan → Executor Agents carry it out
```
One agent plans, others execute. Plan is created upfront based on the task.
